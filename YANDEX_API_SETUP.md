# Инструкция по получению API ключей Яндекс

## Шаг 1: Создание приложения в Яндекс OAuth

1. Перейдите на https://oauth.yandex.ru/
2. Войдите в свой аккаунт Яндекс
3. Нажмите "Зарегистрировать новое приложение"
4. Заполните форму:
   - **Название приложения**: Умный Ассистент (или любое другое)
   - **Платформы**: Web-сервисы
   - **Redirect URI**: `http://localhost:8000/auth/yandex/callback`
   - **Доступ к данным**: 
     - ✅ Доступ к календарю
     - ✅ Доступ к почте
5. Нажмите "Создать приложение"

## Шаг 2: Получение Client ID и Client Secret

После создания приложения вы получите:
- **Client ID** (ID приложения) - скопируйте его
- **Client Secret** (Пароль приложения) - скопируйте его

## Шаг 3: Настройка прав доступа

В настройках приложения убедитесь, что включены следующие права (scopes):
- `calendar:read` - чтение календаря
- `calendar:write` - создание/изменение событий
- `mail:read` - чтение писем
- `mail:write` - отправка писем

## Шаг 4: Добавление в .env файл

Скопируйте полученные значения в файл `.env`:

```env
YANDEX_CALENDAR_CLIENT_ID=ваш_client_id
YANDEX_CALENDAR_CLIENT_SECRET=ваш_client_secret
YANDEX_EMAIL_CLIENT_ID=ваш_client_id  # Можно использовать тот же
YANDEX_EMAIL_CLIENT_SECRET=ваш_client_secret  # Можно использовать тот же
```

## Важные замечания

1. **Redirect URI** должен точно совпадать с тем, что указан в приложении
2. Для продакшена измените Redirect URI на ваш домен
3. Client Secret храните в секрете, не коммитьте в Git
4. Один Client ID можно использовать для обоих сервисов (Calendar и Email)

## Тестирование

После настройки вы сможете авторизоваться через Яндекс OAuth в приложении.
